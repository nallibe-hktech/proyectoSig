1. WHAT IS SIG-es?
================================================================================

A financial operations platform consolidating payroll, invoicing, and operational
data from 10 source systems. Handles complex multi-variable calculations, strict
approval workflows, comprehensive audit trails (5-year fiscal hold).

TARGET USERS: Finance (FICO), Operations, HR, Project Managers
GEOGRAPHY: Spain (fiscal compliance required)
SCALE: ~25 concurrent users, ~500 employees, 20+ active projects

3. PAYMENT CALCULATION MODELS
================================================================================

SIG supports 5+ DISTINCT payment models (each client has 1-3 models):

MODEL A: Monthly Fixed + Variables (Granini, JDE, Inpost)
  = Fixed Salary + (Visits × Bonus) + (Days × Dieta) + (KM × Rate) - Deductions

MODEL B: Hourly (OPTIMISING, Morrison, Cosmetica)
  = (Hours × €11.92/h) + (Extra Hours × OT Rate) + Incentives - Deductions

MODEL C: Per-Unit/Furniture (COTY, DJI, Kobo, Molins)
  = Fixed per Unit + Surcharges for 2nd/3rd Visit + Hours Extra - Deductions

MODEL D: Daily Rate + Incentives (DAIKIN, ITC, Apple)
  = (Contract Salary / Working Days) + Performance Bonus - Deductions

MODEL E: Mixed (Cheil, Ploom)
  = Fixed + Hours-based + Per-Visit + Logistics

KEY RULES:
  - RETROACTIVITY: Conditional. Tariff changes apply forward-looking; FICO can
    request retroactive recalc with version history stored.
  - DEDUCTIONS PRIORITY: Must define order (IRPF → SS → Embargoes → Loans)
  - MINIMUM SALARY: Blocked if < SMI (Spanish national minimum), requires exception
  - MARGIN ALERT: If < 15%, yellow flag; < 10%, red flag (blocker)

================================================================================
4. INVOICE LOGIC (DIFFERENT FROM PAYROLL)
================================================================================

Billing to clients includes MARKUP (margin):

MODEL A: Monthly Fee (Granini, JDE, Inpost, Daikin)
  = Fixed Fee + Variables (Dietas, KM, Travel) + Logistics Fee

MODEL B: Per-Visit (Apple, Dyson, Amex, Morrison)
  = (Visits × Tariff by Type/Zone) + Logistics (COSTE+MARGEN or Fixed)

MODEL C: Hourly + Action (OPTIMISING, Cosmetica)
  = (Hours × €/h) + (Fee per Action) + Logistics

MODEL D: Quote/Presupuesto (DYSON, Kobo, DJI)
  = Agreed Quote + Logistics (if not included) + Revisit surcharge

IMPORTANT: If invoice exceeds presupuesto, escalated to FICO (not auto-capped).

VAT RULE: 21% on domestic; 0% (exempt) on intra-EU services.

5. APPROVAL WORKFLOW
================================================================================

PARTICIPANTS (Multi-level, Non-Linear):

Gestor / Backoffice / Interlocutor
  ├→ Submit to FICO (multiple users can view/edit simultaneously)
  └→ FICO Reviews
      ├→ APPROVE → Lock & Ready for Export to A3
      └→ REJECT → Return to Draft (unlimited retries allowed)

ESCALATION: If FICO queue > 24h, auto-notify FICO + Dirección

NOTES:
  - No sequential "Dirección approval" step (Dirección only sees audit log)
  - Bulk approval needed (approve 100 closures at once)
  - Email notifications required on state changes
  - All role changes logged to audit trail

6. ROLE PERMISSIONS (8 Roles)
================================================================================

GLOBAL ROLES:
  - Admin          → Full system access (users, roles, config)
  - Dirección      → View/Validate/Approve closures, edit users, see KPIs
  - FICO           → Final approval, see all closures, validate concepts
  - RRHH           → Edit payroll, manage contracts, view audit log
  - Facilitador    → Training, support, see full audit log

PROJECT-LEVEL ROLES:
  - Gestor         → Edit own closure, see own projects, submit to FICO
  - Backoffice     → Data entry, edit own closure, manual reconciliation
  - Auxiliar       → Read-only, supporting backoffice

DATA VISIBILITY MASKING:
  - Salary: Hidden from Auxiliar, Backoffice (lower levels)
  - Employee NIF: Hidden from non-HR roles
  - Expense Receipts: Only Admin, FICO, RRHH (of own employee)
  - Audit Log Full: Only Admin + FICO + Facilitador

8. CRITICAL BUSINESS RULES (Blockeante = Must Fix Before Closure)
================================================================================

BLOCKEANTE (System blocks closure):
  ✗ Employee without valid contract for period
  ✗ Expense without assigned project
  ✗ Visit without assigned resource
  ✗ Tariff missing for client+project combination
  ✗ Salary < SMI without documented exception
  ✗ Margin mismatch with presupuesto
  ✗ Unjustified hours exceeding threshold

ADVERTENCIA (Alert, User Confirms):
  ⚠ Extra hours beyond contract max (overtime approval needed)
  ⚠ Margin 10-15% (yellow flag)
  ⚠ Multiple revisits to same center in short period
  ⚠ Anticipated payments not yet cleared
  ⚠ Logistics cost estimates exceeded

ESCALATION: If error unresolved > 24h, escalate to FICO + Admin

9. A3 INTEGRATION (Output Formats)
================================================================================

A3 INNUVA (Payroll Platform):
  Format: Excel 97-2003 (.xls) [NOT .xlsx]
  Frequency: Monthly after FICO approval
  Required Fields: Company, Imputación, Tipo de Paga, Importe Bruto,
                   SS Trabajador, IRPF, Importe Líquido, Descuentos, KM
  Validation: All contracted employees included, no negative gross, reconcile

A3 ERP (Accounting/Invoicing):
  Format: Excel 2007+ (.xlsx)
  Frequency: Monthly after FICO approval
  Data: Client, Invoice lines (description, hours/units, rate, amount),
        21% VAT (domestic) / 0% (intra-EU), Project ref
  Validation: Line items sum to total, >= 1 line per invoice, no dupes
