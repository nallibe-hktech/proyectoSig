================================================================================
KEY RELATIONSHIPS & CONSTRAINTS
================================================================================

1. CLIENTE → PROYECTO (1:M)
   - 1 client can have many projects
   - Project must have a valid client
   - Cascade delete: deleting client archived (not deleted)

2. PROYECTO → VISITA (1:M)
   - 1 project can have many visits
   - Visit must belong to a project
   - Visit can fail (revisit scheduled)

3. PROYECTO → CONCEPTO (M:M)
   - 1 project can have many concepts
   - 1 concept can apply to many projects
   - Application dates define when concept is active

4. RECURSO → CONTRATO (1:M)
   - 1 employee can have many contracts (over time)
   - Only 1 contract can be active at a time
   - validContract flag indicates current active contract

5. RECURSO → VISITA (1:M)
   - 1 employee can conduct many visits
   - Visit must have assigned resource
   - Visit cannot start if resource has no contract

6. RECURSO → GASTO (1:M)
   - 1 employee can have many expenses
   - Expense must be linked to project
   - Expense > €0

7. USUARIO → ROL (M:1)
   - Many users can have same role
   - But user can have multiple roles (Project-level + Global)
   - Role determines permissions and data visibility

8. CIERRE → PAGO (1:M)
   - 1 closure can have many employee payments
   - Pago cannot be in multiple closures
   - Deleting closure cascades to pagos (not exported)

9. CIERRE → FACTURA (1:M)
   - 1 closure can have many customer invoices
   - Factura cannot be in multiple closures

10. CIERRE → APROBACIÓN (1:M)
    - 1 closure can have many approval records (audit trail)
    - Each approval represents one person's action
    - Aprobación is immutable (audit trail)

11. PERÍODO → CIERRE (1:M)
    - 1 period can have multiple closures (payroll + invoicing separate)
    - Cannot edit period once CLOSED

12. CONCEPTO → DEDUCCIÓN (1:M, implicit)
    - Concepts can include or exclude deductions
    - Deduction priority: IRPF → SS → Embargo → Loan → Other


================================================================================
DATA TYPE SPECIFICATIONS
================================================================================

NUMERIC FIELDS:
  - Salaries, Amounts: DECIMAL(12,2) (€0.00 to €999,999,999.99)
  - Percentages: DECIMAL(5,2) (0.00 to 100.00)
  - Hours: DECIMAL(6,2) (0.00 to 9,999.99 hours per period)
  - Kilometers: DECIMAL(7,2) (0.00 to 9,999,999.99 km)

DATE/TIME FIELDS:
  - Dates: DATE (YYYY-MM-DD)
  - Timestamps: DATETIME (YYYY-MM-DD HH:MM:SS UTC)
  - Time of day: TIME (HH:MM:SS, 24-hour format)

TEXT FIELDS:
  - Names: VARCHAR(255)
  - NIF/VAT: VARCHAR(20) with validation regex (Spain format)
  - Email: VARCHAR(255) with email validation
  - Comments: TEXT (up to 4,000 chars)
  - JSON fields: For flexible metadata (calculation formula, etc.)

BOOLEAN/STATUS FIELDS:
  - Is Active: BOOLEAN (1/0 or true/false)
  - Status: ENUM (defined values: DRAFT, APPROVED, etc.)
  - Payment Type: ENUM (MONTHLY, HOURLY, PER_UNIT, etc.)

FOREIGN KEYS (with ON DELETE actions):
  - Cliente: ON DELETE SET ARCHIVED (never hard delete)
  - Recurso: ON DELETE SET INACTIVE
  - Cierre: ON DELETE RESTRICT (if approved/exported)
  - Período: ON DELETE RESTRICT (if locked)

TYPICAL QUERY PATTERNS & INDEXES
================================================================================

FREQUENTLY QUERIED:
  1. Find closures pending FICO approval
     → Index: (status, periodo_id, created_at DESC)

  2. Get all payments for employee in period
     → Index: (recurso_id, periodo_id)

  3. Get invoices by client and date range
     → Index: (cliente_id, periodo_id, fecha)

  4. Audit trail by entity
     → Index: (entity_type, entity_id, timestamp DESC)

  5. Active contracts for employee
     → Index: (recurso_id, status, fecha_fin DESC)

  6. Visits by project in period
     → Index: (proyecto_id, periodo_id, visit_date)

  7. All closures by usuario and role
     → Index: (usuario_id, rol_id, status, timestamp DESC)


================================================================================
ARCHIVAL & DATA RETENTION STRATEGY
================================================================================

HOT DATA (Current Month + 11 Previous):
  - Location: Azure SQL (primary)
  - Access: Full CRUD + indexes for performance

WARM DATA (Years 1-4 Historical):
  - Location: Azure SQL Archive storage tier
  - Access: Read-only, slower queries expected

COLD DATA (Year 5+ Fiscal Hold):
  - Location: Azure Blob Archive (cheapest tier)
  - Access: Off-system, retrieval requires admin action
  - Retention: Until 5-year fiscal hold expires

DELETION:
  - After 5-year hold: Mark as archived, NOT deleted
  - GDPR Requests: Anonymize (not delete) personal data if deletion requested
  - Audit trail: Never deleted (immutable for compliance)


================================================================================
AUDIT LOG DETAIL CAPTURE
================================================================================

Every change to critical entities must be logged:

ENTITIES TO AUDIT:
  - CIERRE (Closure)
  - PAGO (Payment)
  - FACTURA (Invoice)
  - CONCEPTO (Concept/Formula)
  - USUARIO (User)
  - ROL (Role/Permissions)
  - CONTRATO (Contract)

FIELDS TO CAPTURE:
  - auditId (PK, auto-increment)
  - timestamp (DATETIME, UTC)
  - usuario_id (who made the change)
  - entity_type (CIERRE, PAGO, FACTURA, etc.)
  - entity_id (which record was changed)
  - field_name (which field changed)
  - old_value (before change)
  - new_value (after change)
  - change_reason (user comment or system reason)
  - ip_address (for security)
  - session_id (to group related changes)

IMMUTABILITY:
  - Audit records CANNOT be edited or deleted
  - Can only be read by authorized roles (Admin, FICO, Facilitador)
  - Retention: 5 years minimum per fiscal law


================================================================================
CALCULATED/DERIVED FIELDS (NOT STORED, COMPUTED ON READ)
================================================================================

  - Importe Neto (Net) = Importe Bruto - All Deductions
  - Margin (€) = Invoice Total - Total Cost
  - Margin (%) = Margin / Invoice Total × 100
  - Hours Balance = Hours Contracted - Hours Worked + Hours Absences
  - Days with Activity = DISTINCT(date) of visits + expenses in period
  - Accrued Vacation Days = (Hours Worked / Annual Standard) × Vacation Days per Collective
  - Productivity = Visits per Day / Resource
  - Cost per Visit = Total Cost / Visit Count
  - Invoice per Visit = Invoice Total / Visit Count
